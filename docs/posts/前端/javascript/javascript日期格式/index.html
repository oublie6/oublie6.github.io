JavaScript 日期输入

JavaScript 日期输入格式一般有 3 种类型：

类型 例子
ISO 日期 “2015-03-25”（国际标准）
短日期 “2015 年 3 月 25 日”
长日期 “2015 年 3 月 25 日”或“2015 年 3 月 25 日”

ISO 格式遵循 JavaScript 中的严格标准。

其他格式的定义不是很好，可能是特定于浏览器的。

JavaScript ISO 日期

ISO 8601 是日期和时间表示的国际标准。

ISO 8601 语法 (YYYY-MM-DD) 也是首选的 JavaScript 日期格式：const d = new Date("2015-03-25");

计算的日期将与您的时区相关。根据您所在的时区，上述结果将在 3 月 24 日至 3 月 25 日之间有所不同。

ISO 日期（年和月）

ISO 日期可以在不指定日期的情况下写入 (YYYY-MM)：const d = new Date("2015-03");

ISO 日期（仅年份）

ISO 日期可以不带月份和日期 (YYYY)：const d = new Date("2015");

ISO 日期（日期-时间）

ISO 日期可以添加小时、分钟和秒 (YYYY-MM-DDTHH:MM:SSZ)：const d = new Date("2015-03-25T12:00:00Z");

日期和时间用大写 T 分隔。UTC 时间用大写字母 Z 定义。如果要修改相对于 UTC 的时间，请删除 Z 并添加 +HH:MM 或 -HH:MM 代替：const d = new Date("2015-03-25T12:00:00-06:30");

UTC（世界协调时间）与 GMT（格林威治标准时间）相同。在日期时间字符串中省略 T 或 Z 会在不同的浏览器中给出不同的结果。

时区

设置日期时，不指定时区，JavaScript 将使用浏览器的时区。

获取日期时，不指定时区，结果将转换为浏览器的时区。

换句话说：如果在 GMT（格林威治标准时间）中创建日期/时间，如果用户从美国中部浏览，则日期/时间将转换为 CDT（美国中部夏令时）。

JavaScript 短日期。

短日期使用“MM/DD/YYYY”语法编写，如下所示：const d = new Date("03/25/2015");

在某些浏览器中，没有前导零的月份或天数可能会产生错误：

“YYYY/MM/DD”的行为未定义。一些浏览器会尝试猜测格式。有些会返回 NaN。

“DD-MM-YYYY”的行为也未定义。一些浏览器会尝试猜测格式。有些会返回 NaN。

JavaScript 长日期。

长日期通常用这样的“MMM DD YYYY”语法编写：const d = new Date("Mar 25 2015");

月份和日期可以按任意顺序排列：const d = new Date("25 Mar 2015");

而且，月份可以写成全称（January），也可以缩写成（Jan）：const d = new Date("January 25 2015");,const d = new Date("Jan 25 2015");

逗号被忽略。名称不区分大小写：const d = new Date("JANUARY, 25, 2015");

日期输入 - 解析日期

如果您有一个有效的日期字符串，您可以使用该 Date.parse()方法将其转换为毫秒。

Date.parse()返回日期与 1970 年 1 月 1 日之间的毫秒数：let msec = Date.parse("March 21, 2012");

然后，您可以使用毫秒数将其转换为日期对象：

let msec = Date.parse("March 21, 2012");
const d = new Date(msec);
