主从复制是怎么实现

主从复制共有三种模式：全量复制、基于长连接的命令传播、增量复制。

第一次同步

replicaof命令：

# 服务器 B 执行这条命令
replicaof <服务器 A 的 IP 地址> <服务器 A 的 Redis 端口号>

第二阶段将记录在这三个时间段的写操作命令，写入到 replication buffer 缓冲区里

  • 主服务器生成 RDB 文件期间；
  • 主服务器发送 RDB 文件给从服务器期间；
  • 「从服务器」加载 RDB 文件期间；

第三阶段发送这些写操作

完成第一次同步后，维护一个基于keepalive的TCP长连接，通过这个连接将写操作命令传播给从服务器。称为基于长连接的命令传播

分摊主服务器的压力

从服务器可以有自己的从服务器

增量复制

主服务器环形缓冲区repl_backlog_buffer记录写命令，主服务器传播命令时会添加replication offset参数表示该命令在缓存区的偏移。当网络中断又好了之后就将该值发给主服务器，如果主服务器根据缓存区包含该参数对应的命令，则直接将缓冲区写命令发给从服务器进行增量复制，否则进行全量复制（RDB复制和第一次同步一样）

哨兵

监测主节点是否存活，如果发现主节点挂了，它就会选举一个从节点切换为主节点，实现主从节点故障转移。
